Règles du jeu

Note pour les débutants

Ce jeu, comme la plupart des autres jeux, est organisé en niveaux. Vous commencez au niveau 0 et essayez de le “battre” ou de le “terminer”. Finir un niveau donne des informations sur la manière de commencer le niveau suivant. Les pages de ce site pour “Niveau ” contiennent des informations sur la façon de commencer le niveau X à partir du niveau précédent. Par exemple, la page pour le Niveau 1 contient des informations sur comment passer du Niveau 0 au Niveau 1. Tous les niveaux de ce jeu ont une page sur ce site, et ils sont tous liés dans le menu latéral à gauche de cette page.

Vous rencontrerez de nombreuses situations où vous ne saurez pas quoi faire. Ne paniquez pas ! Ne vous découragez pas ! Le but de ce jeu est de vous apprendre les bases. Apprendre les bases inclut la lecture de nombreuses informations nouvelles. Si vous n'avez jamais utilisé la ligne de commande auparavant, une bonne première lecture serait cette introduction aux commandes utilisateur.

Il y a plusieurs choses que vous pouvez essayer lorsque vous ne savez pas comment continuer :

  • Si vous connaissez une commande mais ne savez pas comment l'utiliser, essayez le manuel (page man) en entrant man <commande>. Par exemple, man ls pour apprendre la commande “ls”. La commande “man” a aussi son propre manuel, essayez-le ! Lorsque vous utilisez man, appuyez sur q pour quitter (vous pouvez également utiliser /, n et N pour rechercher).
  • Si la commande n'a pas de page man, elle pourrait être une commande intégrée au shell. Dans ce cas, utilisez la commande help <X>. Par exemple, help cd.
  • Votre moteur de recherche préféré est aussi votre ami. Apprenez à l'utiliser ! Je recommande Google.
  • Enfin, si vous êtes toujours bloqué, vous pouvez nous rejoindre via le chat.

Vous êtes prêt à commencer ! Commencez par le Niveau 0. Bonne chance !

Note pour les machines virtuelles

Il est possible que vous ne puissiez pas vous connecter à overthewire.org via SSH en raison d'une erreur “broken pipe” lorsque l'adaptateur réseau de la machine virtuelle est configuré en mode NAT. Ajouter la ligne IPQoS throughput dans /etc/ssh/ssh_config devrait résoudre le problème. Si cela ne fonctionne pas, la seule option restante est de changer l’adaptateur en mode “Pont réseau” (Bridged mode).
